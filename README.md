# Dr. Nouf Al-Fawzan - Academic Website

هذا المشروع هو موقع أكاديمي ومحفظة متكاملة (Portfolio + Admin CMS) مبنية باستخدام:
- **Next.js 16** (App Router & Server Actions)
- **React 19**
- **Tailwind CSS V4**
- **Vercel** (كبيئة استضافة مثالية)

---

## 🚀 تشغيل المشروع محلياً (Local Development)

### 1. المتطلبات الأساسية
- **Node.js**: يفضل إصدار `v20.x.x` فما فوق.
- **npm** (مثبت مسبقاً مع Node).

### 2. التثبيت والتشغيل
قم بفتح نافذة الأوامر (Terminal) في مجلد المشروع ونفذ الآتي:
```bash
npm install
npm run dev
```
بمجرد التشغيل، يمكنك زيارة الموقع على الرابط: http://localhost:3000
ولوحة التحكم على الرابط: http://localhost:3000/admin

---

## ☁️ أوامر البناء والنشر (Vercel Deployment)

هذا المشروع مبني ليكون متوافقاً بشكل عام (Native) مع بيئة **Vercel**. لا يحتاج المشروع لأي ملفات إعداد إضافية (مثل `vercel.json`) لأن Vercel يكتشف Next.js تلقائياً ويهيئ جميع البيئات اللازمة.

### خطوات الرفع السريع على Vercel (Hobby Plan):
1. ارفع هذا المجلد إلى مستودع خاص بك على Github (كمستودع Private).
2. افتح حسابك في منصة Vercel واضغط على **"Add New Project"**.
3. قم باستيراد (Import) المشروع من Github.
4. وافق على الإعدادات الافتراضية للتثبيت وانقر على **"Deploy"**.

---

## 🔒 إعداد قاعدة البيانات (Environment Variables)

للانتقال من النسخة التجريبية الحالية إلى نسخة إنتاجية حقيقية تعمل على السحابة مستقبلاً، ستقوم بنسخ مفاتيحك وإضافتها في إعدادات مشروعك في Vercel تحت قسم (Settings -> Environment Variables).
المفاتيح تجدها مدونة كأمثلة داخل ملف `.env.example`.

---

## 🚧 Known Limitations / Vercel Serverless Behavior (قيود هامة جداً لنسخة المراجعة)

النسخة المرفقة بين يديك صُممت كنسخة **(Demo / Review Stable)** تعمل بدون قاعدة بيانات خارجية، وتعتمد على محرك وهمي يقرأ البيانات من ملف `db.json` محلياً داخل النظام. 

عند رفع هذه النسخة على Vercel، ستعمل كالتالي:
1. **الموقع العام سيعرض البيانات كالمعتاد (تعمل بنجاح 100%)**: جميع صفحات المراجعة (تفاصيل المقالات، الأبحاث، البحث، وغيرها) ستُقرأ من ملف الـ json وتُعرض للزوار بشكل مثالي لاختبار الواجهة.
2. **⚠️ تعطل الحفظ في لوحة التحكم (مهم)**: استضافة Vercel تعمل بنظام "الخوادم عديمة الخادم" (Serverless Functions) حيث تكون بيئة الملفات فيها **(Read-Only System / بيئة للقراءة فقط)**. هذا يعني أنك أينما أضفت أو عدلت مقالاً من لوحة التحكم في موقع Vercel المنشور، سيظهر لك "خطأ" لأن محرك Json المحلي يحاول حفظ البيانات بالكتابة على الملف ولكن Vercel سيقوم بمنعه كإجراء أمني صلب. **(يجب إتمام المرحلة السحابية لتمكين الكتابة الحقيقية)**.
3. **تجاوز المصادقة**: لوحة التحكم ستفتح وتعمل بدون طلب بيانات دخول لتسهيل جولة المراجعة.
4. **نموذج الاتصال**: غير موصول بخادم بريدي حقيقي وسيظهر نجاحاً وهمياً فقط.

*بُنيت هذه النسخة بشغف لتطابق هويتك الأكاديمية تماماً، وهي الآن مهيئة للإبحار في سحابة Vercel.*
